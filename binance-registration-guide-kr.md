## 4. 2단계 — 계정 생성

가입 페이지에서 원하는 가입 방법을 선택하세요:

### 방법 A — 이메일로 가입

1. **이메일 주소**를 입력합니다.
2. **강력한 비밀번호**를 설정합니다 (영문 대소문자, 숫자, 특수문자 포함 8자 이상).
3. 레퍼럴 코드 입력란에 **`BNBEN`** 이 표시되어 있는지 확인합니다.
4. 바이낸스 이용약관 및 개인정보처리방침에 동의하는 체크박스를 선택합니다.
5. **"개인 계정 만들기"** 버튼을 클릭합니다.

### 방법 B — 휴대폰 번호로 가입

1. 드롭다운에서 **국가 코드**를 선택합니다 (한국: +82).
2. **휴대폰 번호**를 입력합니다.
3. **강력한 비밀번호**를 설정합니다.
4. 레퍼럴 코드 입력란에 **`BNBEN`** 이 표시되어 있는지 확인합니다.
5. 이용약관에 동의 후 **"개인 계정 만들기"** 버튼을 클릭합니다.

### 방법 C — Google / Apple 계정으로 가입

1. **"Google로 계속하기"** 또는 **"Apple로 계속하기"** 버튼을 클릭합니다.
2. 해당 계정으로 인증합니다.
3. 이메일이 자동 입력되면, 레퍼럴 코드란에 **`BNBEN`** 이 입력되어 있는지 확인합니다.
4. 나머지 항목을 작성하고 진행합니다.

## 6. 4단계 — 2단계 인증(2FA) 설정

입금 전에 2FA를 설정하는 것을 **강력히 권장**합니다.

1. 대시보드에서 **프로필 → 보안**으로 이동합니다.
2. **"2단계 인증"** 항목에서 인증 앱 옆의 **"활성화"** 를 클릭합니다.
3. 휴대폰에 **Google Authenticator** 또는 **Binance Authenticator**를 설치합니다.
4. 화면에 표시된 **QR 코드**를 인증 앱으로 스캔합니다.
5. 표시된 **백업 키**를 저장합니다 — 오프라인의 안전한 곳에 보관하세요.
6. 인증 앱에서 생성된 **6자리 코드**를 입력하여 확인합니다.
7. **"활성화"** 를 클릭합니다.

> 🔐 **보안 팁:** QR 코드나 백업 키를 스크린샷으로 찍어 클라우드(Google 포토, iCloud 등)에 저장하지 마세요. 종이에 적거나 암호화된 오프라인 저장소에 보관하세요.

## 7. 5단계 — KYC 본인 인증

바이낸스는 전체 거래 및 출금 한도를 이용하기 위해 본인 인증(KYC)을 요구합니다.

### 인증 단계별 한도

| 단계 | 요구 사항 | 일일 출금 한도 |
|---|---|---|
| 미인증 | 없음 | 제한적 / 없음 |
| 기본 인증 | 신분증 + 셀피 | 최대 200 BTC |
| 고급 인증 | 주소 인증 | 더 높은 한도 |

### 기본 KYC 인증 방법

1. **프로필 → 신원 확인**으로 이동합니다.
2. **거주 국가**를 선택합니다.
3. **신분증 종류**를 선택합니다: 여권, 주민등록증, 운전면허증.
4. 신분증 앞면(및 해당 시 뒷면)의 **선명한 사진**을 업로드합니다.
5. **얼굴 인증**(카메라를 이용한 짧은 안면 스캔)을 완료합니다.
6. 제출 후 승인을 기다립니다 — 보통 **수 분에서 최대 24시간** 소요됩니다.

> 📸 **사진 촬영 팁:** 신분증 전체가 프레임 안에 들어오고, 흔들림 없이 밝은 조명 아래에서 촬영하세요. 빛 반사나 가장자리 잘림은 반려의 주요 원인입니다.

## 8. 6단계 — 첫 번째 입금

인증 완료 후 계정에 자금을 입금할 수 있습니다.

### 방법 A — 암호화폐 입금 (속도 면에서 권장)

1. **지갑 → 법정화폐 및 현물**로 이동합니다.
2. 입금할 암호화폐(예: USDT, BTC) 옆의 **"입금"** 을 클릭합니다.
3. 올바른 **네트워크**를 선택합니다 (예: BEP20, TRC20, ERC20 — 전송 거래소와 반드시 일치시켜야 합니다).
4. **입금 주소**를 복사하여 외부 지갑에서 전송합니다.
5. 네트워크 혼잡도에 따라 보통 **10~30분** 내에 도착합니다.

### 방법 B — 법정화폐로 암호화폐 구매

1. 상단 메뉴에서 **"암호화폐 구매"** 로 이동합니다.
2. **사용할 법정화폐**와 구매할 암호화폐를 선택합니다.
3. 결제 방법을 선택합니다: **카드**, **계좌이체**, 또는 **P2P**.
4. 금액을 입력하고 화면 안내에 따라 진행합니다.

> 💳 **참고:** 카드 결제는 즉시 처리되지만 수수료가 높을 수 있습니다. 계좌이체는 수수료가 낮지만 처리에 1~5 영업일이 소요됩니다.

## 9. 현물 거래 튜토리얼

현물(Spot) 거래는 암호화폐를 실시간 시장 가격으로 직접 사고파는 가장 기본적인 거래 방식입니다. 매수한 코인은 즉시 내 지갑에 보관됩니다.

### 현물 거래 화면 이동

1. 상단 메뉴에서 **거래 → 현물**을 클릭합니다.
2. 좌측 상단 검색창에서 원하는 거래쌍을 입력합니다 (예: `BTC/USDT`, `ETH/USDT`).

### 현물 거래 화면 구성

| 구성 요소 | 설명 |
|---|---|
| 차트 영역 | 캔들스틱 가격 차트 (TradingView 기반) |
| 오더북 | 현재 매수/매도 호가 목록 |
| 거래 내역 | 최근 체결된 거래 목록 |
| 주문 입력창 | 매수/매도 주문 입력 |
| 내 주문 | 미체결 주문 및 체결 내역 확인 |

### 주문 유형

#### 📌 시장가 주문 (Market Order)
현재 시장 가격으로 즉시 체결되는 주문입니다.

1. **"매수"** 또는 **"매도"** 탭을 선택합니다.
2. 주문 유형에서 **"시장가"** 를 선택합니다.
3. 구매할 **금액(USDT)** 또는 **수량**을 입력합니다.
4. **"BTC 매수"** 또는 **"BTC 매도"** 버튼을 클릭합니다.

> ⚡ 시장가 주문은 즉시 체결되지만, 변동성이 큰 시장에서는 예상 가격과 체결 가격에 차이(슬리피지)가 발생할 수 있습니다.

#### 📌 지정가 주문 (Limit Order)
원하는 가격을 직접 지정하여, 시장이 해당 가격에 도달했을 때 체결되는 주문입니다.

1. 주문 유형에서 **"지정가"** 를 선택합니다.
2. **원하는 가격**을 입력합니다 (예: BTC를 60,000 USDT에 매수).
3. **수량**을 입력합니다.
4. **"BTC 매수"** 버튼을 클릭합니다.
5. 주문이 **미체결 주문** 목록에 등록되며, 시장 가격이 지정가에 도달하면 자동 체결됩니다.

> 💡 지정가 주문은 원하는 가격에 매수/매도할 수 있어 장기 투자자에게 적합합니다.

#### 📌 스톱-리밋 주문 (Stop-Limit Order)
특정 가격(스톱 가격)에 도달하면 지정가 주문이 자동 발동됩니다. 손절매(Stop Loss)에 주로 사용됩니다.

1. 주문 유형에서 **"스톱-리밋"** 을 선택합니다.
2. **스톱 가격**: 주문이 발동되는 트리거 가격을 입력합니다.
3. **리밋 가격**: 실제 체결될 지정가를 입력합니다.
4. **수량**을 입력하고 주문을 제출합니다.

| 수수료 유형 | 기본 수수료 | 코드 `BNBEN` 적용 시 |

| 메이커 (지정가 오더 제공) | 0.1% | **0.08%** |

## 10. 선물(합약) 거래 튜토리얼

> ⚠️ **위험 경고:** 선물 거래는 레버리지를 사용하며, 손실이 원금을 초과할 수 있습니다. 충분한 학습 후 소액으로 시작하고 반드시 손절매(Stop Loss)를 설정하세요.

선물(Futures) 거래는 레버리지를 활용하여 암호화폐 가격 변동에 베팅하는 파생상품 거래입니다. 가격 상승 시 이익을 보는 **롱(Long)** 과 가격 하락 시 이익을 보는 **숏(Short)** 포지션 모두 가능합니다.

### 선물 거래 종류

| 종류 | 정산 방식 | 특징 |
|---|---|---|
| USDⓈ-M 선물 | USDT로 정산 | 초보자에게 권장, 가장 많이 사용 |
| COIN-M 선물 | 해당 코인으로 정산 | 코인 채굴자 등에게 적합 |

### 선물 거래 계정 활성화

1. 상단 메뉴에서 **파생상품 → USDⓈ-M 선물**을 클릭합니다.
2. **"선물 개설"** 버튼을 클릭합니다.
3. 위험 고지 안내를 읽고 동의합니다.
4. 선물 계정이 활성화됩니다.

### 선물 계정으로 자금 이체

1. 선물 거래 화면 우측 상단의 **"이체"** 를 클릭합니다.
2. 현물 지갑에서 선물 지갑으로 원하는 금액(USDT)을 이체합니다.
3. **"확인"** 을 클릭합니다.

### 레버리지 설정

1. 거래 화면에서 현재 레버리지 배수(예: `20x`)를 클릭합니다.
2. 슬라이더 또는 직접 입력으로 원하는 레버리지를 설정합니다.
3. **"확인"** 을 클릭합니다.

| 레버리지 수준 | 권장 대상 |
|---|---|
| 1x ~ 5x | **초보자 권장** |
| 5x ~ 10x | 중급자 |
| 10x 이상 | 고급자 (고위험) |

### 마진 모드 선택

| 모드 | 설명 | 권장 대상 |
|---|---|---|
| 격리 마진 (Isolated) | 해당 포지션에만 지정 마진 사용 — 해당 마진만 잃을 수 있음 | **초보자 권장** |
| 교차 마진 (Cross) | 전체 지갑 잔액을 담보로 사용 — 청산 시 전체 잔액 위험 | 경험자 |

### 롱(Long) 포지션 진입 — 가격 상승에 베팅

1. 거래쌍을 선택합니다 (예: `BTCUSDT`).
2. 레버리지와 마진 모드를 설정합니다.
3. 주문 유형을 선택합니다 (시장가 또는 지정가).
4. **매수/롱** 탭에서 금액 또는 수량을 입력합니다.
5. **"매수/롱"** 버튼을 클릭합니다.
6. 포지션 탭에서 진입가, 청산가(Liq. Price), 미실현 손익(uPNL)을 확인합니다.

### 숏(Short) 포지션 진입 — 가격 하락에 베팅

1. **매도/숏** 탭을 선택합니다.
2. 금액 또는 수량을 입력합니다.
3. **"매도/숏"** 버튼을 클릭합니다.

### 손절매(Stop Loss) 및 이익실현(Take Profit) 설정

포지션 진입 후 반드시 TP/SL을 설정하세요:

1. **포지션** 탭에서 보유 중인 포지션을 찾습니다.
2. **"TP/SL"** 버튼을 클릭합니다.
3. **이익실현 가격 (Take Profit)**: 목표 수익을 실현할 가격을 입력합니다.
4. **손절매 가격 (Stop Loss)**: 손실을 제한할 가격을 입력합니다.
5. **"확인"** 을 클릭합니다.

> 🔴 **청산 가격(Liquidation Price)을 항상 확인하세요.** 손절매는 청산 가격보다 충분히 여유 있게 설정해야 합니다. 선물 거래에서 손절매 없이 거래하는 것은 매우 위험합니다.

### 포지션 청산 (포지션 닫기)

1. **포지션** 탭에서 닫을 포지션을 찾습니다.
2. **"시장가 청산"** 클릭 시 즉시 포지션이 종료됩니다.
3. 특정 가격에 닫으려면 **"지정가 청산"** 을 선택합니다.

### 선물 거래 핵심 용어 정리

## 12. 자주 묻는 질문 (FAQ)

**Q: 바이낸스 가입은 무료인가요?**  
A: 네, 가입은 완전 무료입니다.

**Q: 20% 환급이란 정확히 무엇인가요?**  
A: 레퍼럴 코드 `BNBEN`으로 가입하면 기본 수수료 대비 20%가 영구적으로 할인됩니다. 예를 들어, 현물 거래 수수료가 0.1%에서 0.08%로 낮아집니다.

**Q: 현물 거래와 선물 거래의 차이는 무엇인가요?**  
A: 현물 거래는 암호화폐를 실제로 사고파는 방식으로, 매수한 코인을 직접 보유합니다. 선물 거래는 레버리지를 활용해 보유 자산보다 큰 금액을 거래하는 파생상품으로, 수익이 극대화될 수 있지만 그만큼 손실 위험도 큽니다.

**Q: 선물 거래는 초보자도 할 수 있나요?**  
A: 가능하지만 권장하지 않습니다. 현물 거래로 충분히 경험을 쌓은 후, 바이낸스 **테스트넷(모의 거래)** 으로 연습하고 소액으로 시작하세요.

**Q: 청산(Liquidation)이란 무엇인가요?**  
A: 레버리지 선물 포지션에서 손실이 담보(마진)를 초과하면 시스템이 자동으로 포지션을 강제 청산합니다. 격리 마진 모드에서는 해당 포지션의 마진만 손실되고, 교차 마진에서는 전체 잔액이 위험에 처할 수 있습니다.

**Q: KYC 인증은 필수인가요?**  
A: 입금, 출금, 전체 거래 기능을 이용하려면 KYC 인증이 필요합니다. 미인증 계정은 이용 한도가 크게 제한됩니다.

**Q: 모바일에서도 이용할 수 있나요?**  
A: 네, 공식 **바이낸스 앱**을 [App Store](https://apps.apple.com) 또는 [Google Play](https://play.google.com)에서 다운로드하여 현물 및 선물 거래 모두 이용 가능합니다.

**Q: 가입할 때 레퍼럴 코드를 입력하지 않았어요. 나중에 적용할 수 있나요?**  
A: 아쉽게도 레퍼럴 코드는 가입 시에만 적용됩니다. 이미 가입한 후에는 소급 적용이 불가능합니다.

**Q: 바이낸스에 자산을 보관하는 것이 안전한가요?**  
A: 바이낸스는 **SAFU(사용자 자산 보호 펀드)** 를 운영하여 비상 상황 시 사용자 자산을 보호합니다. 그러나 장기 보유 자산은 개인 하드웨어 지갑에 보관하는 것이 모범 사례입니다.
